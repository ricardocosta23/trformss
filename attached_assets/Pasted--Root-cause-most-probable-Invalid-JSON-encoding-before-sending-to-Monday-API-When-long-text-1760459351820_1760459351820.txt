🎯 Root cause (most probable)

👉 Invalid JSON encoding before sending to Monday API

When long_text answers include quotes, newlines, or special characters, you must wrap the entire dict with json.dumps(column_values) before including it in your GraphQL string.

If you insert the dict directly (e.g. column_values={column_values}), Python will produce single quotes ' instead of ", and Monday’s GraphQL parser will reject the mutation.

Example of what fails:

mutation = f'''
mutation {{
  create_item(
    board_id: {board_id},
    item_name: "{item_name}",
    column_values: {column_values}
  ) {{
    id
  }}
}}
'''


Output in request:

column_values: {'long_text_mksdw43g': 'ITA, boa companhia ...'}


❌ Invalid JSON (uses single quotes)

What you must do:

import json

mutation = f'''
mutation {{
  create_item(
    board_id: {board_id},
    item_name: "{item_name}",
    column_values: {json.dumps(column_values)}
  ) {{
    id
  }}
}}
'''


Output in request:

column_values: {"long_text_mksdw43g": "ITA, boa companhia ..."}


✅ Valid JSON
